## Purpose

This case records lessons from a practical TUYA Elementor build that used an older prompt.

It is a case memory, not a fixed architecture template.

---

## Lessons

```text
Use smaller steps when visual risk is high.
Do not treat visual estimates as confirmed values.
Prefer official Elementor docs for standard capability claims.
Use current UI evidence for executable control paths.
Keep meaningful content in normal flow.
Use overlay stages only for controlled visual layers.
Classify style reuse before choosing Variable, Global Class, Local Class, Component, or CSS.
Build one repeated item, verify it, then duplicate it.
Preserve unresolved responsive unknowns.
```
